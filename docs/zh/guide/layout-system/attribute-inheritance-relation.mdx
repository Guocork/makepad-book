# 属性继承和覆盖机制

一个重要的概念是布局属性的继承和覆盖机制

## Walk 属性不继承

- 每个组件都需要明确定义自己的 Walk 属性
- 子组件的 Walk 完全独立于父组件

## Layout 属性部分继承

- flow 方向会影响子组件的默认行为
- spacing 会应用于同级子组件之间
- align 会影响所有子组件的对齐方式

```rust
// Layout 属性继承示例
Parent = <View> {
    layout: {
        flow: Down,
        align: {x: 0.5} // 所有子组件默认水平居中
    },

    <Child1> {
        // 继承父级的 flow 和 align
    }

    <Child2> {
        layout: {
            align: {x: 1.0} // 覆盖继承的对齐方式，改为右对齐
        }
    }
}
```
